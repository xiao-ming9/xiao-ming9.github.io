阿里实习总结

发表于 2020-07-26 更新于 2020-08-01 阅读次数：

春招

今年的春招刚开始的时候其实并不是很顺利。记得最先投的一份简历应该是大三上学期末投了 CVTE（其实还不算春招），那是第一次面试，不过二面之后也就销声匿迹了。春招开始后投的第一家是字节，也是最先面的一家，当时投的是一个 go 开发岗，一面的感觉不是很好，有两个题答不出来，由于是正式的第一次面试，也比较紧张，最后果不其然就凉了。

本来最想进的是腾讯，在腾讯刚开始春招的当天晚上就投了简历。然而很遗憾的是迟迟没有收到面试通知，期间选的 IEG 打过一次电话说是先了解一下，但是了解完就没有下文了。在阿里开始春招前陆续还投了百度、网易、美团等。其中美团也是面了一面就凉了，网易因为到了后面有点心灰意冷就没有去参加笔试了，百度则等到拿了阿里 offer 才开始面，是一个移动应用开发岗。

说回阿里的面试，阿里从提前批就开始投了，不过提前批很遗憾并没有通过。正式批开始后流程走的很快，那天正在吃晚饭，面试官就打了电话过来（入职后发现是个准备升 P8 的大佬），很尴尬的约了一个小时后，面试的过程也非常欢快，面试官非常耐心，印象中最深的就是回答一个问题，讲着讲着自己都有点乱了，他仔细听了之后给了一个“嗯，我懂你意思”的肯定，瞬间就把我从紧张的心情中拉了回来，回答问题也更加有感觉。

二面的面试官电话里的声音特别好听（一开始以为是个20来岁的小哥哥，入职后发现原来是自己的团队领导），他讲的非常耐心，面试期间对于问题的提问也是一步一步提出，渐渐深入，再慢慢讨论，有一说一，阿里的面试体验是所有公司面试里面个人感觉最棒的。

三面是在二面的一个星期后，二面的面试官还特地打电话说明了一下情况。面试官入职后有点对不上号是哪一个人，面了 50 分钟左右，过程同样是一个很轻松的氛围。

四面则是一个交叉面，问了特别多计网的知识，当时一度觉得有点悬。

五面其实就是 HRG 面了，后面因为说笔试成绩不是很好，所以在 HRG 面通过之后又加了一面，说是补笔试，但其实就做了一道算法题，其余还是问问题，面试的是整条线的大 leader。

走完所有流程后其实等了很久才等来了 offer，卡在 HRG 那卡了一个多月左右（明白可能是在当备胎）。期间还面了腾讯的 PCG，几乎是体验最最最最差的一次面试，本来很开始捞我面试了，非常兴奋，结果面试过程回答问题不断被否定，最后还问了些特别特别刁钻的问题（还是自己太菜），面完自己心态崩了一晚上。后来还面了一次 IEG，本来感觉还不错，但是也挂了。

入职

有一说一，阿里的实习生待遇非常不错，包酒店 14 天住宿，本来一开始订的阿里旗下的菲斯布渴，但是到了杭州说是没房子了，改到了维也纳，环境其实也很不错，而且每天有 90 元洗衣服务（用自助洗衣机一次只要 15），提供免费早餐。另外还提供来回车费发票报销，不管是飞机票、机场打车到酒店的费用统统报销。另外每天还有 13 元餐补，晚上 6 点后吃晚饭再补贴 13 元，晚上 9 点后还有 10 元夜宵券，并且免费打车回家。

和女朋友一起提早一周去了杭州，一边玩一边找房子租。

入职当天因为疫情原因在酒店进行了入职培训，不过培训后就去了园区找师兄拿了电脑，然后在那里无所事事呆了一天，主要是配置了电脑环境，下午四五点就归心似箭了（女朋友还在酒店等我呢）。

实习生活

在我们团队的三个实习生中，我是最晚入职的一个，所以刚入职后我主要是先学习了集团使用的中间件的使用，包括 Pandora（Spring 改造），Pandora Boot（Spring Boot），MetaQ（对应开源的 RocketMQ），Tair（分布式缓存数据库），TDDL（分布式数据库，底层MySql），HSF（远程调用服务），Diamond（分布式全局配置中心），Config Server（分布式注册中心）等。

进去之后没几天，师兄们便布置了一个数据质量运维的小需求给了我们，因为我刚进来什么也不会，所以一开始基本上都是其他两个实习生在讨论，而我则埋头学我的中间件，过了一周感觉差不多了，才开始对接需求，负责其中标签质量运维平台的开发。

阿里是弹性上班，实习这段时间基本上是每天早上九点到公司，因为租的房子稍微有点远，所以是 8 点起床，然后洗脸刷牙完就跑去搭公交了，晚上一般九点后才走，因为打车免费，而且还可以白嫖 10 块钱的夜宵。

园区里有健身房（没去过，没什么时间），还有推拿室（长这么大第一次体验推拿，挺舒服），吃的食堂基本就近，吃过五号楼和七号楼的，以及园区里的餐厅弄堂里。

跟其他两个实习生小伙伴也日渐熟络。

在阿里参加的集体活动最大的一个应该就是 WIN FAST KO 大会，早上开会听大佬讲成就和未来目标，下午开启了环绕西湖大暴走（9.8公里），小组队员还拿下了第二名（队长挺优秀）。

另外一个就是 P10 座谈会，P10 大佬整条线的人坐在一起，听听大佬来阿里一路经历的风风雨雨，并且就一些职场问题进行答疑。

其他的日常基本就是打码，学习。接下来就学习的内容做一个总结。

技术总结

业务需求

在阿里做的最主要的业务是对内部的数据质量运维平台的开发。我主要负责的是标签质量模块的开发。简单来说，就是显示当日的标签质量，并进行红黑榜排序。

关于标签的概念：淘宝对于其使用的用户都会标识上若干个标签来划分人群。有些标签是根据用户信息生成的，例如：“性别”；有些标签是根据用户在淘宝的购买行为生成的，例如“低消费人群”等等；还有一种标签则是通过商家进行特定的圈选之后形成的一类人群标签，例如根据“性别”，“低消费人群”，“某商品应用人群”等多个标签筛选出来的人群可以形成一个新的标签。同时，标签下还有对应的值，比如说“性别”对应的值是男、女。另外值的类型也多种多样，有可能是枚举，也有可能的一个不固定的时间，还有可能是无限枚举、嵌套枚举等等。

使用标签划分出来的人群，可以用于千人千面、主动营销和人群联动。

在阿里的大数据平台，采用分库分表的形式进行数据存储，表中的数据每天都有产出，按照时间分区，当天产出的放在当天的 ds 分区下，另外，对于标签相关的表，会有二级分区，二级分区以 tag_part 即标签进行划分。我做的功能主要就是针对这些表，监控其每天 tag_part 分区下的数据产出情况，从而判断标签的健康情况。

判断标签健康情况的标准主要有：

  • 当天数据是否产出
  • 当天数据产出是否延迟（下午4点后产出）
  • 产出的数据中的记录数较前一天，前三天的数据波动量大小

对于产出延迟或者连续未产出的数据，应当进行告警促达。

业务代码

代码中最复杂的部门主要有两个，一个是对于健康分的计算，另一个是页面的展示。

健康分计算

健康分的计算根据前面提到的标准进行，在其中进行复杂度计算的时候，由于计算中涉及到方差计算波动值，在处理的时候，一开始用了方差公式：

并且对于变量使用 Long 类型，之所以使用这条，是因为可以实现针对产生的分区数据只遍历一次就计算出结果。

实际开发过程中，发现分区下的记录数有一些在平方之后超过了 Long 的范围，导致最后的方差结果变成负数，在开方之后会出现 NaN 异常。对于这种情况，采用了 BigDecimal 进行数据的处理，这里注意到的一个问题是，对于 BigDicimal 的等值判断，不能使用 == 和 equals，在这种情况下像 0 和 0.0 会认为是不相等的，通过源码可以发现原因在于其 equals 判断会去判断整数部分和 scale 即小数点部分，所以认为不相等。对于 BigDecimal 的判断，应当采用 compareTo 方法。

页面展示

页面展示出于用户体验的需要，需要将当日已更新的数据放在前面，而当日未更新的数据则放在后面并进行置灰，所以需要利用子查询，对每一标签都先根据 max(gmt_modified) 查询出最新更新时间，再根据更新时间进行排序，这里通过建立索引进行优化。

另一方面，由于采用 PageHelper 插件，使用了静态的 ThreadLocal 参数，分页参数和线程是绑定的。在使用过程中发现可能存在线程污染的问题，会对没有使用 PageHelper 的 sql 语句在同一线程内也加上了 LIMIT 关键字，造成查询出错。为了安全起见，最后每次都手动进行了 PageHelper.clearPage()。

前端代码

由于整个平台需要从 0 到 1 自己搭建，所以还需要自己设计前端页面，之前没有接触过，所以在这里其实抓着工作室的前端同学问了特别多。稍微记录一下收获。

前端采用的是集团内部的中台框架 ice，本质上用的还是 react。在 React 中 setState 为异步执行，它会合并多个 setState 在一段时间后执行，所以一开始通过 setState 方法来传递红黑榜参数的 type 发现总是不一致，最后通过 useEffect 监听 type 解决问题

数据库

数据库的设计思路比较简单，主要一个就是针对数据库的一些字段建立索引（在这里可以手动指定索引，不走优化器判断），另外一个就是保证数据的幂等性。

索引

对每天用于红黑榜排序的 score 字段建立索引，对 （tag_info_id，gmt_modified)建立联合索引，用于查看详情以及七天使用情况查询，根据最左前缀原则，常用的放左边，还可以单独用作索引。

幂等性设计

对于数据的落库，会先执行数据库查询，防止当天数据的重复插入。

技术架构学习

技术架构的学习主要是针对中间件的学习，等有空再一篇一篇整理。

再出发

在阿里实习了两个月的时间，然后腾讯又再一次把我捞了起来，这一次的面试流程特别快，当天晚上进行了两轮面试，一周之内就拿了 offer（可能是听到在阿里实习，加上可能赶指标吧hh），不过因为自己本来就想去的鹅厂，自己和家里人也都觉得杭州实在是太远了，所以最后决定还是离职。写这篇总结的时候，人也刚到杭州，在酒店吃完晚饭没事就总结一下。聚是一团火，散是满天星，杭州，再会！腾讯，我来啦！
